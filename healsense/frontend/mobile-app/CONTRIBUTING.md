# Contributing to HealSense Mobile

Thanks for your interest in contributing! As this is my Final Year Project, I really appreciate any feedback or suggestions.

## How to Contribute

### Reporting Issues
If you find a bug or have a suggestion:
1. Check if it's already been reported in the Issues section
2. If not, create a new issue with:
   - Clear title and description
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior
   - Screenshots if applicable

### Suggesting Features
I'm open to feature suggestions! Please:
- Explain the use case
- Describe how it would help users
- Provide examples if possible

### Code Style
I'm following these conventions:
- TypeScript for type safety
- Functional components with hooks
- Meaningful variable and function names
- Comments for complex logic
- Dark mode support for all screens

### Testing
Before submitting changes:
- Test on both light and dark modes
- Verify on different screen sizes
- Check that alerts and vital signs work correctly
- Ensure authentication flow is smooth

## Development Setup

1. Fork the repository
2. Clone your fork
3. Install dependencies: `npm install --legacy-peer-deps`
4. Create a feature branch: `git checkout -b feature/your-feature-name`
5. Make your changes
6. Test thoroughly
7. Commit with clear messages
8. Push to your fork
9. Create a Pull Request

## Questions?

Feel free to open an issue for any questions. I'm still learning, so constructive feedback is always welcome!
